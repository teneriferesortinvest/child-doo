# UX Review — child-doo — 2026-05-13

**Build URL:** `file:///Users/tinomachristi/Desktop/child-doo/index.html#contact`
**Builder version reviewed:** v0 (per BRIEF.md §Status)
**Reviewer:** studio-ux (Agent Smith Studio)

## Verdict
**Ship-with-fixes — blocked on UX for Spanish-primary audience.** The page is visually finished and the interactions work, but the build delivers an English-shell to a Spanish-primary buyer with no phone CTA, a fake form, and a contact lead that addresses already-booked customers. Fix the language posture, contact channel, and form copy before this can credibly convert a Tenerife parent.

## User flow trace (3 journeys)

### Journey 1 — Spanish parent looking for a niñera (primary buyer)
- **Expects:** "una página en español, una niñera de confianza, un teléfono."
- **Delivers:** `<html lang="en">` + English headlines + a form that says "Have any questions about your booking" — implying she has *already* booked. No phone number anywhere. The Spanish words that exist are decorative tags under English service names.
- **Friction:** Cognitive load to translate, no immediate trust signal (no founder photo, no testimonials), no phone, mismatched lead microcopy.
- **Verdict:** This is the buyer the brief targets and she gets the worst experience.

### Journey 2 — English-speaking relocator / holidaymaker
- **Expects:** Quick scan, "do they have last-minute babysitters in Tenerife?"
- **Delivers:** Headline answers it ("we connect parents with amazing child-care professionals in the south of tenerife"). `Emergency cover` service card is visible. But still no phone — only a form.
- **Friction:** Holidaymaker wanting urgent care expects a phone/WhatsApp tap, not a form-and-wait. BRIEF §5 has phone as an open decision; placeholder OK but a "WhatsApp coming soon" stub would soften the gap.
- **Verdict:** Works directionally, fails on conversion path for the *urgent* segment.

### Journey 3 — Sponsor/partner (e.g. hotel concierge, relocation agency)
- **Expects:** "Are these people legit? Can I refer guests to them?"
- **Delivers:** "Our story" section gives Melania's credentials, vetting process listed in Advantages. No partnership CTA, no LocalBusiness verification badge, no photo of Melania.
- **Friction:** Trust signal density is low. No press, no founder image, no testimonials, no Instagram handle (BRIEF doesn't specify one but social proof is missing).
- **Verdict:** Partial — story is told, proof is absent.

## Affordance audit

| Element | Affordance signal | Verdict |
|---|---|---|
| `nav.top a` | `:hover` underline-reveal animation | **weak on desktop, invisible on touch.** No persistent affordance — link looks like body text until hovered. |
| Hand-drawn logo (SVG) | `cursor:pointer`, `:hover` wiggle | **false affordance.** Looks clickable, nothing happens on click. Either wire it to `#hero`/top-of-page, or remove `cursor:pointer`. |
| Baby photo (`baby.jpg`) | `cursor:pointer`, `:hover` spin | **false affordance.** Same issue — clickable cursor, no handler. |
| Service card | `:hover` lift + green border + tint | **clear on desktop, invisible on touch.** Cards have no role / no link target — they're decorative containers, not CTAs. A first-time visitor will try to click them. |
| Pillars (Confianza / Fiabilidad / Seguridad) | none | **correct** — these are signage, not interactive. |
| Advantage rows | none | **correct.** |
| Step circles (1/2/3) | none | **correct.** |
| `cta-pill` "Necesito asesoría →" | dark pill, `:hover` lift | **clear.** The strongest CTA on the page. |
| Form submit button | dark button, `:hover` scale | **clear**, but no `:focus-visible` styling — keyboard users get the browser default outline overridden by `outline:none` on inputs (button retains default, but inputs lose it). |
| "Tell us what you need" inline link in pricing-note | dashed-green underline | **clear and well-styled.** |
| Heart (♥) glyph in form corner | none | **decorative but reads as a button.** Pink + top-right corner is the universal location for a close-X. First-time user *may* try to dismiss. |

## Interaction quality findings

- **Form labels:** `<label>` elements wrap inputs but contain **no visible or screen-reader text** — only `placeholder` attributes. Screen readers will announce the placeholder via aria-placeholder, but VoiceOver still says "edit text" before reading it. Add `aria-label` per input or visible `<span>` inside the label.
- **Focus management:** `input:focus` recolours the border (good). `button:focus` has no rule — relies on browser default. Inputs override `outline:none` indirectly via `:focus` but no `:focus-visible` fallback for keyboard users. Tab-only flow is usable but the focused state is faint.
- **No modal exists** — so focus-trap / Esc-to-close N/A. The "modal" here is `window.alert()`, which is jarring 1990s UX. Replace with inline success message (e.g. swap the form body for a "Gracias — te contactamos en X horas" panel).
- **Audio cleanup:** N/A (no media).
- **Touch targets (measured from CSS, viewport ≤760px):**
  - `nav.top a` — `font-size:13px` + `padding:4px 0` ≈ **21px tall × link-width**. ❌ **Fails WCAG 2.5.5 (44×44)** and Apple HIG (44pt).
  - `.cta-pill` — `padding:14px 28px` + 16px font ≈ **~44px tall**. ✅ borderline-pass.
  - Form `button` — `padding:12px 24px` + 14px font ≈ **~38px tall**. ⚠️ under 44px.
  - Form `input` — `padding:12px 14px` + 14px font ≈ **~38px tall**. ⚠️ under 44px (acceptable for inputs since precision tap is on a 100%-wide row).
  - Service card — full card is the tap area (~140px tall) but card has **no link role**, so the large target is wasted.
- **Hover-only affordances flagged for touch:**
  - Logo wiggle (decorative — keep, OK)
  - Service card lift + green border (the hover-only state IS the affordance — broken on touch)
  - Nav underline reveal (the only nav affordance — broken on touch)
  - Baby photo spin (decorative, but combined with `cursor:pointer` it implies an action)

## Mobile-specific findings (≤760px breakpoint)

- **Nav stacking:** `flex-direction:column` + `align-items:flex-start`. The first `<div>` contains brand-mini AND "contact us." on stacked rows; the other two links (`our services`, `our story`) follow as separate flex items. The visual order ends up: brand-mini → contact us → our services → our story. Works but feels accidental — looks like CSS-driven, not designed.
- **Pillar strip:** `flex-wrap:wrap` with `gap:48px` will likely render 3-column at 320px-portrait, wrapping to 2+1 at ~340–400px because each pillar has `min-width:120px`. Acceptable.
- **Service grid:** `minmax(220px, 1fr)` — single column on iPhone SE (375px usable after padding), two-column from ~520px. ✅
- **Hero image (`baby.jpg`):** `width:min(320px, 68vw)` — well bounded. ✅
- **No horizontal scroll detected** in the CSS (max-widths and `box-sizing:border-box` set globally). ✅
- **Perceived performance:** The SVG logo wipe-in plays for ~3.45s before the brand is fully drawn. **First Meaningful Paint of the brand is delayed.** A returning visitor will see the same animation every load — annoying. Use `prefers-reduced-motion` to skip the animation and a one-time `sessionStorage` flag for subsequent visits.

## Cognitive load findings

- **5-second test (Spanish parent):** Logo → baby → English tagline. After 5s the question "what is this site, in my language" is **half-answered.** The Spanish-speaking buyer must translate "we connect parents with amazing child-care professionals" to confirm the offer.
- **5-second test (English visitor):** Logo → baby → English tagline. **Fully answered.** But the brand mark renders gradually over 3+ seconds, slowing comprehension.
- **Information density:** Hero is spacious (good). Service grid + advantages + steps stack into a long page — total ~6 vertical sections. No table of contents / jump-bar in nav (only 3 of 5 sections exposed) means a scanner can't jump to "Why us" or "How it works" from the top.
- **Forced decisions:** None aggressive. Pricing is deferred to form contact, which is honest but adds friction. A "desde €X / hora" anchor (BRIEF §5 open question) would dramatically lower commitment cost.
- **Trust signal density:** **LOW.** No founder photo, no testimonials, no review stars, no LocalBusiness badge visible (schema.org JSON-LD exists but is invisible), no phone, no physical address, no "número de niñeras en la red". For a service that touches one of the highest-trust categories there is — leaving children with strangers — this page does not yet earn that trust.

## Action items for Builder v2 (severity-ranked)

### Blockers (ship-stopping UX defects for the Spanish-primary audience)

1. **Set `<html lang="es">`** (currently `en`) and ship Spanish primary copy for all section titles, hero tagline, advantage labels, form lead, and CTAs. Keep English versions for v2 with a toggle, per BRIEF §2 and §5.
2. **Fix the form lead copy.** `"Have any questions about your booking or feedback about our service?"` addresses an existing customer. Replace with first-touch copy: e.g. *"Cuéntanos qué necesitas — tipo de servicio, fechas, edad de los peques — y te respondemos en menos de 24 h."*
3. **Add a phone / WhatsApp CTA** in the nav and/or sticky footer. Even a placeholder *"WhatsApp pronto"* is better than the current silence given the `niñera express` (emergency) service. BRIEF §5 has the number as an open decision — flag it to Tino / Melania, don't ship without.
4. **Add the Phone field** to the contact form (BRIEF §4.4 explicitly recommended), with `inputmode="tel"` and optional country prefix.
5. **Replace `alert('Thanks!')`** with an inline success state (swap form body to a confirmation panel) and wire the form to a real endpoint or note the form is non-functional v0.

### Major (visible but not ship-stopping)

6. **Nav completeness:** Add `#why` and `#how` to the top nav. Five page sections, only three in nav is inconsistent.
7. **Remove `cursor:pointer` from logo and baby photo** (or wire them — logo→top, baby→`#story`). False affordance.
8. **Add a founder photo** in the "our story" section. Childcare is a trust category; a face from Melania is worth more than any other element on this page.
9. **Increase nav-link tap-target** to ≥44px tall on mobile (currently ~21px). Add vertical padding in the mobile media query.
10. **Add visible/`aria-label` text to form `<label>` elements** so screen readers don't say "edit text" before the placeholder.
11. **Replace the decorative ♥ in the form corner** or move it inline — its current position reads as a close button.
12. **Honor `prefers-reduced-motion`** on the logo wipe-in, baby-bob, and service-card lift animations.

### Minor (polish)

13. **Add a `<h1>`** for the visible brand mark or hero tagline. Currently there is no `<h1>` on the page (the SVG carries `aria-label="child-doo"` but no semantic `<h1>`).
14. **Normalise brand case** — `<title>` says `child-doo` (lowercase), BRIEF says `CHILD DOO` (uppercase). Pick one for the title tag.
15. **Pricing anchor** — add *"desde €X / hora"* somewhere visible (BRIEF §5 flagged this as SEO + CTR positive).
16. **Skip-to-content link** for keyboard users (`<a href="#services" class="skip-link">`).
17. **`:focus-visible` ring** on the submit button and nav links for keyboard navigation.
18. **Service cards:** decide if they are interactive or decorative. If interactive, wrap each in a link to a `/servicios/<slug>` page or a `#contact?service=<slug>` deeplink. If decorative, remove the hover-lift (which implies clickability).
19. **`section.pillars` has no `id`** — give it `id="valores"` so it can be linked from nav and external traffic.
20. **OG locale:** `og:locale` is `en_GB` with alternate `es_ES` — invert these once primary copy is Spanish.

---

**Reviewer's note:** The build is craft-loving and visually warm. The gap is **audience-fit**: it reads like a portfolio piece by a designer, not a service site by Melania for a Spanish parent. Fixing 1–5 above moves it from "nice draft" to "credibly converts the buyer in BRIEF §2".
